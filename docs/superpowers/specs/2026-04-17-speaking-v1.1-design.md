# Speaking Coach v1.1 — History, Random Topic, UI Redesign

## Context

The Speaking Coach feature (v1) is implemented and working. Users can pick a curated or custom topic, record/type a response, get AI-powered corrections, and hear natural alternatives via TTS. Results are persisted and synced cross-device.

Three improvements are needed:
1. **History sessions** — view and revisit past speaking practice results
2. **Random topic** — quick-start without browsing the topic list
3. **Home screen redesign** — better layout with clear visual hierarchy

## Decisions

| Decision | Choice | Reason |
|---|---|---|
| History detail level | Full result view with TTS playback | Most value for review; transcript captures what user said |
| Original audio storage | Not stored | Too heavy for sync; transcript is sufficient |
| Random topic interaction | Bottom sheet with shuffle + confirm | Avoids landing on uninteresting topics; shuffle feels playful |
| Home screen priority | Balanced — quick-start first, progress second, browse third | Random topic is the most common action; history shows progress without dominating |
| Schema changes | None | Existing `speaking_results` table has all needed fields |

## Home Screen Redesign

Layout (top to bottom):

1. **AppBar**: "Speaking Practice" title + history clock icon button
2. **Random Topic card**: Gradient card with dice icon, "Start with a surprise topic" subtitle. Tap opens bottom sheet.
3. **Custom topic input**: Text field with arrow submit (unchanged behavior)
4. **Recent Practice**: Section header + "See all" link. Horizontal scroll of last 5 sessions. Each card: topic (2-line truncated), color-coded correction badge (red 2+, green 0-1), relative time. Hidden if no history. Tappable → history detail.
5. **Browse Topics**: Section header, ExpansionTiles by category with topic count (e.g. "8 topics").

### Random Topic Bottom Sheet

Triggered by tapping the Random Topic card:
- Category label (e.g. "Daily Life")
- Topic text
- Two buttons: **Shuffle** (pick another random) + **Start Practice** (navigate to record screen)
- Swipe down to dismiss

## History

### History List Screen

Reached via clock icon in home AppBar or "See all" from recent practice section.

- AppBar: back button + "History" title
- Vertical list, most recent first
- Each item: topic text, correction count badge (color-coded), relative time, chevron
- Custom topics: additional blue "Custom" badge
- Swipe-to-delete: soft delete + undo snackbar
- Empty state: centered mic icon + "No practice sessions yet"

### History Detail Screen

Reached by tapping a history list item.

- AppBar: "Results" title + red "Delete" text button
- Topic label at top
- Same layout as post-session result screen:
  - Transcript card
  - Overall note (if present)
  - Natural version with TTS play button
  - Corrections list with per-correction TTS play buttons
- No "Practice another topic" / "Done" buttons — view-only

## Data Layer Changes

No schema or migration changes. The existing `speaking_results` table already stores all needed data and syncs cross-device.

New code:
- `SpeakingService.getResultById(String id)` — fetch single row by ID
- Expand `SpeakingHistoryItem` to include `correctionsCount`
- `speakingResultByIdProvider(String id)` — FutureProvider.family, loads row from DB, parses into `SpeakingResult`
